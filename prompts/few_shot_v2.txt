## Example 3: Mixed Bottleneck (Reduction Pattern)

### Code:
```fortran
! Fortran-style reduction with conditional
subroutine reduce_conditional(a, mask, n, result)
    real(8), intent(in) :: a(n)
    logical, intent(in) :: mask(n)
    integer, intent(in) :: n
    real(8), intent(out) :: result
    
    integer :: i
    real(8) :: local_sum
    
    local_sum = 0.0d0
    do i = 1, n
        if (mask(i)) then
            local_sum = local_sum + a(i) * a(i)
        end if
    end do
    result = local_sum
end subroutine
```

## Analysis Guidelines

When analyzing code, consider:
1. **Hotspot identification**: Look for nested loops, expensive operations (division, transcendentals), and high iteration counts
2. **Bottleneck classification**: Calculate or estimate arithmetic intensity (FLOPs/byte). >10 is compute-bound, <1 is memory-bound, 1-10 depends on hardware
3. **Memory patterns**: Sequential (good), strided (moderate), indirect/random (poor)
4. **Language patterns**: Recognize both C/C++ and Fortran idioms
5. **GPU suitability**: Consider parallelism, memory access patterns, and data transfer overhead
